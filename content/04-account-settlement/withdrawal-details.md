## Definition

```
GET https://api.nihaopay.com/v1.2/withdrawal/detail/{withdrawal_id}
```

> **Quirk:** this is a *single-record detail* endpoint that returns **CSV**, not JSON. Most "detail" endpoints return a single JSON object. Plan parsing accordingly.

Returns the full data for a specific withdrawal — typically each transaction the withdrawal aggregated.

## Sample request

```bash
curl https://api.nihaopay.com/v1.2/withdrawal/detail/{withdrawal_id} \
  -H "Authorization: Bearer <TOKEN>"
```

## Request parameters

| Property | Required | Description |
|---|---|---|
| `withdrawal_id` | Required (URL) | The ID generated by the original `Apply Withdrawal` request, also visible in [Withdrawal history](./withdrawal-history.md). |

## Response

`Content-Type: text/csv`. The CSV concludes all the data associated with this withdrawal — typically one row per underlying transaction that was rolled up.

The column shape isn't documented in the v1.2 spec.
